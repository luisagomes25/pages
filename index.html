O que é o Sensorium?

O Sensorium é uma plataforma completa para monitoramento inteligente de ambientes, integrando sensores físicos (ESP32, Arduino, Raspberry Pi Pico) a um backend robusto em Python (FastAPI), com visualização de dados em dashboards web modernos. Ideal para aplicações acadêmicas, industriais e residenciais.

FastAPI MySQL Docker Pytest Arduino/ESP32 Raspberry Pi Pico HTML5 CSS3 JavaScript JWT OAuth2

Principais Funcionalidades

Gerenciamento de Usuários

Sistema completo de cadastro e login com autenticação JWT, validação de senhas e proteção de rotas.

Controle de Acesso e Permissões

Associação de sensores a usuários específicos, controle de acesso granular e diferentes níveis de permissão.

Visualização de Dados em Tempo Real

Dashboards interativos para visualização de dados dos sensores, incluindo pH, umidade e nível de água.

Gerenciamento de Sensores

Interface administrativa para gerenciamento de sensores e atribuição de sensores a usuários.

Notificações

Sistema de notificações para alertas e eventos importantes.

Configurações e Regras de Alerta

Configurações personalizáveis e regras de alerta baseadas em leituras de sensores.

API Restful

Nossa API fornece endpoints para:

  • Autenticação (login, cadastro, recuperação de senha)
  • Gerenciamento de usuários e sensores
  • Registro e recuperação de leituras de sensores
  • Gerenciamento de notificações
  • Criação e configuração de regras de alerta

Acesse a documentação completa da API em: http://localhost:8001/docs (durante execução local)

Tecnologias Utilizadas

Análise completa do projeto:

Backend

  • Linguagem: Python
  • Framework Principal: FastAPI (para construção de APIs de alta performance)
  • Validação de Dados: Pydantic (para validação, serialização e documentação automática)
  • ORM: SQLAlchemy (comunicação com MySQL)
  • Autenticação e Segurança:
    • JWT (tokens de acesso para proteger rotas)
    • OAuth2 (fluxo de autenticação, incluindo Login com Google)
    • BCrypt (hashing de senhas)
  • Envio de E-mails: Yagmail (notificações e confirmações)
  • Servidor ASGI: Uvicorn (execução de alta performance da aplicação)

Frontend

  • Estrutura e Estilo: HTML5, CSS3, Bootstrap 5
  • Interatividade e Visualização de Dados: JavaScript, Chart.js
  • Renderização no Servidor: Jinja2

Banco de Dados

  • Principal: MySQL 8.0
  • Testes: SQLite (em memória para testes unitários)

Testes e Qualidade de Código

  • Pytest (automação de testes unitários e de integração)
  • Coverage.py (análise de cobertura de código)
  • Testes de integração para todas as funcionalidades principais

DevOps e Infraestrutura

  • Docker e Docker Compose (containerização da aplicação e dependências)
  • Caddy (proxy reverso e gerenciamento de SSL)
  • Execução de testes automatizados antes do início do servidor

Hardware e Comunicação

  • Microcontroladores: ESP32, Arduino, Raspberry Pi Pico W
  • Sensores: Nível de água (boia), pH, umidade do solo
  • Protocolos: HTTP/HTTPS (comunicação web e hardware), Serial (Arduino)

Como Executar Localmente

  • Clone o repositório principal e execute docker-compose up --build
  • Acesse http://localhost:8001/ para visualizar o sistema
  • A documentação da API está disponível em http://localhost:8001/docs

Arquitetura do Banco de Dados

Nossa arquitetura de banco de dados inclui as seguintes tabelas principais:

  • Usuarios: Armazena informações de usuários (nome, email, senha hash, endereço)
  • Locais: Representa sensores/dispositivos (nome, tipo, descrição, chave API)
  • Leituras: Registros de dados dos sensores (pH, umidade, nível de água)
  • Notificações: Mensagens e alertas para usuários
  • Configurações: Parâmetros configuráveis do sistema
  • Regras de Alerta: Condições que acionam notificações
  • Usuário-Sensor: Tabela de associação para definir quais sensores cada usuário pode acessar
