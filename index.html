Sensorium

Monitoramento inteligente e integração de sensores em tempo real.

O que é o Sensorium?

O Sensorium é uma plataforma completa para monitoramento inteligente de ambientes, integrando sensores físicos (ESP32, Arduino, Raspberry Pi Pico) a um backend robusto em Python (FastAPI), com visualização de dados em dashboards web modernos. Ideal para aplicações acadêmicas, industriais e residenciais.

FastAPI MySQL Docker Pytest Arduino/ESP32 Raspberry Pi Pico JWT/OAuth2 Chart.js

Principais Funcionalidades

Gerenciamento de Usuários

Sistema completo de cadastro e login com autenticação JWT, validação de senhas e proteção de rotas.

Controle de Acesso e Permissões

Associação de sensores a usuários específicos, controle de acesso granular e diferentes níveis de permissão.

Visualização de Dados em Tempo Real

Dashboards interativos para visualização de dados dos sensores, incluindo pH, umidade e nível de água.

Gerenciamento de Sensores

Interface administrativa para gerenciamento e atribuição de sensores a usuários.

Notificações e Alertas

Sistema de notificações e regras de alerta baseadas em leituras de sensores para eventos importantes.

API Restful

Nossa API fornece endpoints robustos para a gestão completa do sistema:

  • Autenticação (login, cadastro, recuperação de senha)
  • Gerenciamento de usuários e sensores
  • Registro e recuperação de leituras de sensores
  • Gerenciamento de notificações e regras de alerta

Tecnologias Utilizadas

Análise completa das ferramentas que tornam o Sensorium possível:

Backend

  • Linguagem: Python
  • Framework Principal: FastAPI (APIs de alta performance)
  • ORM/DB: SQLAlchemy (com MySQL)
  • Segurança: JWT, OAuth2 (Login com Google) e BCrypt (hashing de senhas)
  • Servidor ASGI: Uvicorn

Frontend

  • Estrutura: HTML5, CSS3, Bootstrap 5
  • Visualização: JavaScript e Chart.js (gráficos interativos)
  • Renderização: Jinja2

DevOps e Infraestrutura

  • Containerização: Docker e Docker Compose
  • Testes: Pytest (Unitários e de Integração) com Coverage.py
  • Proxy: Caddy (Proxy reverso e SSL)

Hardware e Comunicação

  • Microcontroladores: ESP32, Arduino, Raspberry Pi Pico W
  • Sensores: Nível de água (boia), pH, umidade do solo
  • Protocolos: HTTP/HTTPS (comunicação web e hardware), Serial (Arduino)

Como Executar Localmente

Para executar o sistema completo (API + Frontend + Banco de Dados), você precisa do Docker instalado. Note que esta página é estática e não executa a API.

  • Clone o repositório principal: git clone https://github.com/Gmarques07/Sensorium-UFRPE.git
  • Navegue até o diretório e execute: docker compose -f docker-compose.dev.yml up --build
  • Acesse o sistema em: http://localhost:8001/
  • A documentação da API (Swagger UI) estará disponível em: http://localhost:8001/docs

Como Executar na Web

  • Acesse o sistema online através do link: https://sensoriumtech.online/

Arquitetura do Banco de Dados

O sistema é construído sobre as seguintes tabelas principais:

  • **Usuarios**: Armazena informações de usuário e autenticação (e.g., nome, email, senha hash).
  • **Locais**: Representa sensores/dispositivos (nome, tipo, descrição, chave API).
  • **Leituras**: Registros de dados dos sensores (pH, umidade, nível de água, timestamp).
  • **Notificações**: Mensagens e alertas para usuários.
  • **Regras de Alerta**: Condições que acionam notificações (e.g., se pH < 6.0, enviar alerta).
  • **Usuário-Sensor**: Tabela de relacionamento para controle de acesso (quem pode ver qual sensor).
